このハンズオンでは、Googleのノーコードアプリ開発ツール「AppSheet」を使って、Googleスプレッドシートから業務アプリを作成する方法を学びます。

コードを1行も書かずに、スマートフォンで使えるアプリが10分で完成します。

このコースで学ぶこと

  • AppSheet の基本概念とできること
  • Googleスプレッドシートからアプリを作成する方法
  • アプリのプレビューと操作
  • データの追加・編集・検索
  • ビュー（画面）のカスタマイズ
  • アプリの共有方法

対象者

  • 「スプレッドシートは使えるけどアプリ開発は未経験」という方
  • 現場スタッフがスマホで入力できる業務ツールを作りたい方
  • プログラミング経験不要

所要時間

約 45分

使用するもの

  • Google アカウント
  • Googleスプレッドシート（このコースで作成します）
  • AppSheet（appsheet.com）— Google Workspace に含まれる

AppSheet は Google が提供するノーコードアプリ開発プラットフォームです。スプレッドシートやデータベースをデータソースとして、iOS・Android・Webブラウザで動くアプリを作成できます。

AppSheet でできること

機能 説明
データ入力フォーム 現場スタッフがスマホから直接データを入力できる
一覧・詳細表示 スプレッドシートのデータをリスト・カード・マップ形式で表示
自動化 データ追加時にメール送信・Slack通知などを自動実行
オフライン対応 圏外でも使え、接続時に自動同期
権限管理 ユーザーごとに閲覧・編集・追加の権限を細かく設定

業務での活用例

  • 点検・巡回アプリ：現場でスマホから点検結果を入力、写真も添付。Excelファイルへの手入力が不要に。
  • 備品管理アプリ：QRコードをスキャンして在庫確認・貸出管理。
  • 顧客訪問記録：外出先で訪問メモを入力、会社のスプレッドシートにリアルタイム反映。
  • 申請・承認フロー：スマホで申請→担当者に通知→承認のワークフローを自動化。
費用：Google Workspace Business Standard以上に含まれています。個人での試用はフリープランも利用可能（一部機能制限あり）。

AppSheet のデータソースとなるスプレッドシートを作成します。

今回は「社内備品管理アプリ」を作ります。

スプレッドシートを作成する

  1. sheets.google.com を開く
  2. 「新しいスプレッドシート」をクリック
  3. シート名を「備品管理」に変更

ヘッダーとデータを入力する

A1セルから以下のデータを入力してください。

A: ID B: 備品名 C: カテゴリ D: 数量 E: 保管場所 F: 状態
001ノートPCIT機器5情シス倉庫利用可
002プロジェクター会議室23F会議室利用可
003電源タップ電気10情シス倉庫利用可
004HDMIケーブルIT機器32F倉庫利用可
005ホワイトボード会議室4各会議室一部修理中
重要：A列（ID）は重複のない一意の値にしてください。AppSheet がこれをキー（レコードの識別子）として使います。

シートを確認してから次へ

ヘッダー行（1行目）と5件のデータ（2〜6行目）が入力できたら次のステップへ進みます。

スプレッドシートから直接 AppSheet アプリを作成します。

方法1：スプレッドシートから直接作成（推奨）

  1. 作成したスプレッドシートを開く
  2. 上部メニュー「拡張機能」→「AppSheet」→「アプリを作成」をクリック
  3. AppSheet エディターが新しいタブで開く

方法2：AppSheet から作成

  1. appsheet.com を開く
  2. 「作成」→「アプリ」→「既存のデータから始める」をクリック
  3. 「Google スプレッドシート」を選択し、「備品管理」シートを選択

AppSheet エディターの構成

エリア 説明
左メニュー Data（データ）・Views（画面）・Automation（自動化）・Security（権限）の設定
中央パネル 選択した設定の詳細を編集する
右プレビュー スマホ画面のリアルタイムプレビュー。実際の動きを即確認できる

AppSheet がスプレッドシートを自動解析し、列のデータ型（テキスト・数値・ドロップダウンなど）を自動設定します。

自動生成の精度：ヘッダー名が日本語でも英語でも正しく認識されます。数値列は自動で「Number」型に、「状態」のような列は「Enum（選択肢）」型として設定されることがあります。

自動生成されたアプリの操作を確認します。

リスト表示を確認する

右のプレビューに備品一覧がリスト表示されています。各備品をタップすると詳細画面が開きます。

  1. プレビュー画面で「ノートPC」をタップ
  2. 詳細画面（ID・カテゴリ・数量・保管場所・状態）が表示されることを確認
  3. 鉛筆アイコンをタップして編集画面を確認
  4. 戻る（←）ボタンで一覧に戻る

新しいレコードを追加する

  1. プレビューの右下の「＋」ボタンをタップ
  2. 入力フォームが開く
  3. 以下を入力：
    • ID: 006
    • 備品名: マウス
    • カテゴリ: IT機器
    • 数量: 8
    • 保管場所: 情シス倉庫
    • 状態: 利用可
  4. 「保存」をタップ

スプレッドシートへの反映を確認する

  1. スプレッドシートのタブに切り替える
  2. 7行目に「マウス」のデータが追加されていることを確認

アプリからの入力がリアルタイムでスプレッドシートに反映されることが確認できました。

検索機能：プレビュー上部の検索バーに「IT機器」と入力すると、カテゴリが一致する備品だけに絞り込まれます。AppSheet が自動でテキスト検索を用意してくれています。

ビュー（アプリの画面）を追加・カスタマイズして使いやすくします。

カードビューに変更する

  1. 左メニューの「Views」をクリック
  2. 自動生成された「備品管理」ビューをクリック
  3. 「View Type」を「Table」から「Card」に変更
  4. プレビューがカード形式に変わることを確認

カテゴリでグループ化する

  1. 「Sort, Group」タブをクリック
  2. 「Group by」に「カテゴリ」を追加
  3. プレビューで「IT機器」「会議室」「電気」のグループに分かれることを確認

「状態別」ビューを追加する

  1. 「Views」ページで「新しいビューを追加」をクリック
  2. 以下を設定：
    • View Name: 状態別
    • For this data: 備品管理
    • View Type: Deck（カード積み重ね表示）
    • Group by: 状態
  3. 「保存」をクリック

「利用可」「一部修理中」のグループでビューが表示されます。修理中の備品をすぐ把握できるようになりました。

ナビゲーションに追加する

作成したビューはアプリ下部のナビゲーションバーに自動追加されます。プレビューで確認してみましょう。

ビューの種類：Table（表）・Card（カード）・Deck（デッキ）・Gallery（ギャラリー）・Map（地図）・Form（フォーム専用）など多数あります。データの性質に合わせて選びましょう。

完成したアプリを他のメンバーと共有します。

アプリを保存・デプロイする

  1. 右上の「保存」ボタンをクリック
  2. 「アプリ名」を「備品管理アプリ」に変更
  3. 「保存」をクリック

ユーザーを招待する

  1. 左メニューの「Users」をクリック
  2. 「Add Users」をクリック
  3. 共有したいメンバーのメールアドレスを入力
  4. 権限を選択：
    • Editor: 追加・編集・削除が可能
    • Viewer: 閲覧のみ
  5. 「招待を送信」をクリック

スマホでアプリを使う

  1. 招待したメンバーのスマホで受信したメールを開く
  2. リンクをタップするとブラウザでアプリが開く
  3. 「ホーム画面に追加」でスマホのアプリアイコンとして保存できる
AppSheet アプリの共有URL：左上のメニュー「Share」→「Copy app link」で直接URLをコピーしてSlackやメールで共有できます。
注意：スプレッドシートの共有設定も確認してください。AppSheet ユーザーがスプレッドシートにアクセスできる権限がないと、アプリが正しく動作しません。スプレッドシートの「共有」設定で、同じメンバーに閲覧または編集権限を付与してください。
