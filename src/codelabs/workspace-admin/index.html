このコースについて

Google Workspace の管理コンソールを使って、ユーザー管理・グループ管理・セキュリティ設定・レポート確認を行う情報システム担当者向けハンズオンです。

学習目標

  • 管理コンソールの基本操作を理解する
  • ユーザーアカウントの作成・編集・削除を行う
  • グループを作成してメール配信リストを設定する
  • 2段階認証などセキュリティポリシーを設定する
  • 監査ログとレポートで利用状況を把握する

必要なもの

  • Google Workspace 管理者権限のあるアカウント
  • ブラウザ（Google Chrome 推奨）
本番環境での注意
このコースの操作は実際の Workspace 環境に影響します。テスト用ユーザー・グループで練習してから本番作業を行ってください。

管理コンソールへのアクセス

アクセス方法

  1. ブラウザで admin.google.com を開く
  2. 管理者アカウントでログイン
  3. 管理コンソールのダッシュボードが表示される

組織部門（OU）とは

ユーザーをグループ化して、部署ごとに異なるポリシーを適用する仕組みです。

例: 「営業部」OUでは YouTube をブロック、「情報システム部」OUでは全サービスを許可、というように設定できます。

ユーザー管理の基本操作

新しいユーザーを追加する

  1. 管理コンソール → ユーザー → 新しいユーザーを追加
  2. 以下の情報を入力して「追加」をクリック
項目入力例備考
名前田中 太郎日本語OK
メールアドレスtanaka@example.co.jp一意である必要あり
パスワード（自動生成を推奨）初回ログイン時に変更を強制できる
組織部門/営業部後から変更可能
一括追加
多数のユーザーを追加する場合は CSV ファイルでの一括アップロードが便利です。管理コンソール → ユーザー → 「ユーザーを一括アップロード」から実行できます。

ユーザー情報を編集する

  1. 管理コンソール → ユーザー → 対象ユーザーをクリック
  2. 編集したい項目（名前・所属部署・役職など）の右にある鉛筆アイコンをクリック
  3. 変更して「保存」

パスワードをリセットする

  1. ユーザーの詳細画面 → パスワードをリセット
  2. 「次回ログイン時にパスワードを変更するよう求める」にチェック
  3. 「リセット」をクリックして仮パスワードを本人に伝達

ユーザーを一時停止・削除する

削除は慎重に
ユーザーを削除するとメール・ドライブのデータも失われます（20日間は復元可能）。退職者の処理は「一時停止」→「データ移管」→「削除」の順で行うことを推奨します。
  1. ユーザーの詳細画面 → ユーザーを停止 または ユーザーを削除
  2. 削除する場合はデータ移管先ユーザーを指定することを推奨

グループ（メーリングリスト）の管理

グループはメール配信リストや、Google Drive・Chat などへのアクセス権限管理に使います。

グループを作成する

  1. 管理コンソール → グループ → グループを作成
  2. 以下を設定する
項目設定例説明
グループ名全社連絡表示名
グループのメールall@example.co.jp送信先アドレス
グループの説明全社員への連絡用任意
アクセスタイプチーム（メーリングリスト）用途に応じて選択

メンバーを追加する

  1. 作成したグループをクリック → メンバー → メンバーを追加
  2. ユーザー名またはメールアドレスを入力して追加
  3. 役割を「メンバー」「オーナー」「マネージャー」から選択
役割できること
オーナーグループ設定変更・メンバー管理・メッセージ投稿
マネージャーメンバー追加・削除・メッセージ投稿
メンバーグループへのメール送受信のみ

グループを活用する場面

  • 全社配信メール: all@example.co.jp に送るだけで全員に届く
  • 部署グループ: sales@example.co.jp で営業部全員に共有
  • プロジェクトグループ: ドライブフォルダのアクセス権限をグループ単位で設定
  • Gemini / Workspace 機能のアクセス制御
グループのメールアドレスを Google Drive のフォルダ共有設定に追加すると、グループメンバー全員に一括でアクセス権を付与できます。

セキュリティの強化

2段階認証を強制する

  1. 管理コンソール → セキュリティ → 認証 → 2段階認証プロセス
  2. 「2段階認証プロセスの使用を許可する」をオンにする
  3. 「強制」を選択して組織部門（全組織 or 特定部署）を選ぶ
  4. 猶予期間（例: 1週間）を設定して「保存」
猶予期間の設定を忘れずに
即日強制にすると、まだ設定していないユーザーがログインできなくなります。全ユーザーに事前周知してから猶予期間付きで実施してください。

パスワードポリシーを設定する

  1. 管理コンソール → セキュリティ → パスワードの管理
  2. 推奨設定を確認する
設定項目推奨値
最小文字数12文字以上
パスワードの再利用過去3回は禁止
強度の強制強（大文字・数字・記号を含む）
有効期限180日（組織ポリシーに準拠）

外部共有のポリシー

  1. 管理コンソール → アプリ → Google Workspace → ドライブとドキュメント
  2. 共有設定 → 外部共有の範囲を設定
設定内容
オフ外部共有を完全禁止
許可（ドメイン制限）指定ドメインとのみ共有可
許可（リンク共有禁止）招待した外部ユーザーとのみ共有可
許可（全て）リンク共有含め外部共有可

利用状況の把握と問題調査

利用状況レポート

  1. 管理コンソール → レポート → 使用状況
  2. アプリ別（Gmail・Drive・Meet）のアクティブユーザー数を確認
ライセンスを付与しても使っていないユーザーをここで発見できます。未使用ライセンスのコスト最適化に役立てましょう。

監査ログの確認

  1. 管理コンソール → レポート → 監査ログ
  2. 調査したい操作の種類を選択
ログの種類確認できること
管理ログ管理者操作（ユーザー作成・削除・設定変更）
ログイン監査ユーザーのログイン記録・不審なアクセス
Drive 監査ファイルの閲覧・ダウンロード・共有操作
Gmail 監査メール送受信の詳細ログ
Meet 監査会議への参加・退室の記録

セキュリティアラートの設定

  1. 管理コンソール → セキュリティ → アラートセンター
  2. 不審なログイン・マルウェア添付・フィッシングの検出アラートを設定
  3. メールアドレス（情シス担当者）に通知されるよう設定する

アプリのアクセス制御と Gemini の有効化

Google アプリの有効化・無効化

  1. 管理コンソール → アプリ → Google Workspace
  2. 各サービスの横にあるトグルで有効/無効を切り替え
  3. 組織部門ごとに設定を変えることも可能

Gemini の有効化

  1. 管理コンソール → アプリ → Google Workspace → Gemini
  2. 「Gemini の使用を許可する」をオン
  3. 有効にする組織部門を選択して保存
段階的な展開のすすめ
最初は情シス部門や AI Champion メンバーに限定して有効化し、フィードバックを受けてから全社展開する方法が安全です。

サードパーティアプリの管理

  1. 管理コンソール → セキュリティ → API 制御
  2. 承認していないサードパーティアプリが Google データにアクセスしていないか確認
  3. 不審なアプリはここでアクセスを取り消すことができる
