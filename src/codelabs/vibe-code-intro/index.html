Vibe-coding とは

「Vibe-coding（バイブコーディング）」とは、AIに日本語で指示するだけでコードを書かせる開発スタイルです。

自分でコードを一行一行書く代わりに、「こういうものを作りたい」と伝えると、AIが動くコードを生成してくれます。エラーが出ても「これを修正して」と言えばAIが直してくれます。

このコースで体験すること

  • AIへの「指示の出し方」を学ぶ（プロンプトエンジニアリング）
  • Gemini CLI をインストールして対話的にコードを生成する
  • エラーをAIに自律修正させる
  • GASのコードを日本語指示だけで完成させる

前提条件

  • Googleアカウントを持っていること
  • Windowsのコマンドプロンプト（またはMacのターミナル）を開けること
  • 「GAS 4行入門」「GAS 基礎」を完了していること（推奨）

所要時間

約 50 分

コードが書けなくてもOK
このコースのゴールは「AIにどう伝えるか」を学ぶことです。プログラミング経験がなくても完走できます。

従来の開発 vs Vibe-coding

従来の開発 Vibe-coding
作業者 人間がコードを書く AIがコードを書く
必要スキル プログラミング言語の文法知識 何をしたいか言語化する力
エラー対応 自分で原因を調べて修正 AIにエラーを渡して修正させる
完成速度 数時間〜数日 数分〜数十分

Vibe-coding でうまくいく「指示の書き方」

悪い例 ❌

「コードを書いて」

→ 何を作りたいかわからない。汎用的な内容しか返ってこない。

良い例 ✅

「Google Apps Scriptで、スプレッドシートのA列にある住所の一覧を読み込み、
各住所をGoogleマップのURLに変換して、B列に書き出すコードを書いてください。
シート名は「住所リスト」です。」

→ 具体的な入出力・使うサービス・シート名まで伝えている。すぐ動くコードが返ってくる。

良い指示の5要素

  1. 使うサービス：Google Apps Script / Python / Excelマクロ など
  2. 何をするか：読み込む・変換する・送信する・生成する など
  3. 入力データ：どこに・何が入っているか
  4. 出力結果：どこに・どんな形式で書き出すか
  5. 条件・制約：エラー処理・文字数・ファイル名 など

「AIはテレパシーが使えない」
頭の中にある「こういう感じ」をAIは読めません。言語化した量がそのまま完成度に直結します。最初は雑でも、対話を重ねながら精度を上げていきます。

Gemini CLI とは

Gemini CLIはGoogleが提供するコマンドラインAIツールです。ターミナルで日本語を打つだけでコード生成・ファイル操作・エラー修正ができます。

無料枠あり（Googleアカウントのみ。ただし無料枠ではデータが学習に使われる可能性があります）。

インストール手順

事前準備：Node.js のインストール確認

Gemini CLIの動作にはNode.jsが必要です。まずインストール済みか確認します。

Windows の場合:

  1. スタートメニューで「cmd」と検索してコマンドプロンプトを開く
  2. 以下を入力してEnter：
node --version

バージョン番号（例: v20.11.0）が表示されればOK。

「コマンドが見つかりません」と出た場合は nodejs.org からLTS版をインストールしてください。

Mac の場合:

  1. Spotlight（Cmd+Space）で「ターミナル」を開く
  2. node --version を実行して確認

Gemini CLI のインストール

npm install -g @google/gemini-cli

インストール完了後、バージョン確認：

gemini --version

ログイン

gemini

初回起動時にブラウザが開き、Googleアカウントでの認証を求められます。認証を完了するとターミナルに戻り、対話モードが始まります。

社内PCでの注意
社内PCにnpmのインストール権限がない場合は、IT部門に相談するか、個人PCで試してください。Google AI Studioの ウェブ版 でも同様の体験ができます。

Gemini CLI で GAS コードを生成する

Gemini を起動してターミナルで日本語の指示を打ち込みましょう。

実習 1：シンプルな指示

ターミナルで gemini を起動後、以下を入力してください：

Google Apps Scriptで、アクティブなスプレッドシートのA列に入力された
メールアドレスの一覧を読み取り、各アドレスに「テスト送信です」という
件名でメールを送信するコードを書いてください。
ヘッダー行（1行目）はスキップしてください。

生成されたコードをコピーして、Apps Scriptエディタに貼り付けて動かしてみましょう。

実習 2：反復改善（イテレーション）

コードを見て「ここを変えたい」と思ったら、追加で指示します：

さらに修正してください：
- 送信前に「X件のメールを送信します。続けますか？」と確認ダイアログを出す
- 送信済みのアドレスはB列に「送信済み」と記録する
- エラーが出たアドレスはB列に「エラー」と記録する

会話として積み重ねる
Gemini CLIは前の会話を覚えています。「さらに〇〇してください」と続けることで、コードを段階的に育てられます。一度に完璧なコードを求めず、少しずつ精度を上げる方が実用的です。

実習 3：業務に近い指示を書く

以下のようなケースで自由に試してみましょう：

Google Apps Scriptで、毎朝9時に自動実行される関数を作ってください。
処理内容：
- 「タスク管理」という名前のスプレッドシートを開く
- B列「期限」が今日以前で、C列「状態」が「未完了」の行を検索する
- 該当する行のA列「タスク名」を一覧にして、
  自分のGmailアドレスに「期限切れタスク：〇件」という件名でメールを送る

このように「業務の流れをそのまま言語化する」だけでコードができます。

エラーは「AIへの追加情報」

生成されたコードをそのまま動かすと、エラーが出ることがあります。これはよくあることです。エラーメッセージをそのままAIに渡すだけで、自律的に修正してくれます。

エラー修正の手順

ステップ 1：エラーメッセージをコピーする

Apps Scriptエディタでエラーが出た場合：

  1. 「実行」ログに表示されるエラーメッセージ全体をコピー
  2. 例：TypeError: Cannot read properties of undefined (reading 'getRange')

ステップ 2：AIに渡す

Gemini CLIに以下の形式で投げます：

以下のエラーが出ました。修正してください：

エラー: TypeError: Cannot read properties of undefined (reading 'getRange')
行番号: 5

現在のコード:
function sendEmails() {
  var sheet = SpreadsheetApp.getActiveSpreadsheet();
  var range = sheet.getRange("A2:A100");
  ...
}

ステップ 3：修正版を受け取って差し替える

AIが修正版コードを返してくれます。元のコードと差し替えて再実行します。

エラーとうまく付き合う
エラーが出ることは「失敗」ではありません。AIとの対話を通じてコードが完成に近づくプロセスです。1回で動かなくても、2〜3回のやり取りで大抵のエラーは解消されます。

よくあるGASのエラーと対処法

エラー よくある原因 AIへの指示例
Authorization required 権限の承認が必要 「初回実行時の権限承認の手順を教えて」
Exception: Service invoked too many times API呼び出しが制限を超えた 「レート制限を回避するよう修正して」
ReferenceError: xxx is not defined 変数名のタイポ エラーメッセージをそのまま渡す
TypeError: Cannot read properties of undefined 存在しないセル・シートへのアクセス エラーメッセージとコードをそのまま渡す

AIと対話してGmail Add-onを完成させる

Gmail Add-onとは、Gmailの画面にサイドパネルとして表示できる小さなアプリです。メールを開いたときに自動で情報を表示したり、ボタンを押すと処理が走ったりします。

実習：メール要約アドオンを作る

「メールを開いたときにGeminiが自動で要約を表示するアドオン」を作ります。

ステップ 1：Gemini CLIに指示する

Google Apps ScriptでGmail Add-onを作ってください。
要件：
- Gmailでメールを開くとサイドパネルに要約が表示される
- 要約はGemini API（モデル: gemini-1.5-flash）で生成する
- APIキーはスクリプトプロパティ「GOOGLE_API_KEY」から取得する
- 要約は200文字以内の日本語で生成する
- サイドパネルには「要約」と「元のメール件名」を表示する

ステップ 2：生成されたコードを確認する

AIが生成するコードには通常以下の要素が含まれます：

  • appsscript.json（マニフェスト）：アドオンのスコープ・ホームページの設定
  • Code.gs：メール取得・Gemini API呼び出し・カード表示のロジック

典型的なコード例（AIが生成するものの参考）：

function buildAddOn(event) {
  var accessToken = event.messageMetadata.accessToken;
  var messageId = event.messageMetadata.messageId;

  GmailApp.setCurrentMessageAccessToken(accessToken);
  var message = GmailApp.getMessageById(messageId);
  var subject = message.getSubject();
  var body = message.getPlainBody().substring(0, 2000);

  var summary = callGemini(
    "以下のメールを200文字以内で日本語で要約してください：\n\n" + body
  );

  var card = CardService.newCardBuilder()
    .setHeader(CardService.newCardHeader().setTitle("AI 要約"))
    .addSection(
      CardService.newCardSection()
        .addWidget(CardService.newTextParagraph().setText("件名: " + subject))
        .addWidget(CardService.newTextParagraph().setText(summary))
    )
    .build();

  return [card];
}

ステップ 3：テストデプロイ

  1. Apps Scriptエディタで「デプロイ」→「テストデプロイ」をクリック
  2. 「インストール」を選択
  3. Gmailを開き、任意のメールをクリック
  4. 右側のサイドパネルにアドオンのアイコンが表示されることを確認

Gmailの内容について
このアドオンはメール本文をGemini APIに送信します。社内の機密メールには使用しないでください。テスト用の個人メールで動作確認してください。
