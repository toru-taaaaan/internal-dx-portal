このCodelabでは Looker でデータを可視化するための一連のフローを解説します。BigQueryとの接続・認証、LookMLモデルの作成、Exploreでの可視化まで、ゼロから順番に手を動かして学べます。

出典：本教本は g-gen テックブログ「LookerでBigQueryに接続してLookML作成から可視化までを行う手順を解説」（著：菊池 健太、2026年2月18日公開）に基づいています。

このCodelabで学ぶこと

  • Lookerの基本構成要素（ビュー・Explore・モデル）
  • BigQuery側のデータセット作成とサービスアカウント設定
  • LookerからBigQueryへの接続（Connection）の作成
  • LookMLプロジェクトの自動生成と調整方法
  • ディメンションとメジャーの違い・修正方法
  • 日付データを時系列で扱うための dimension_group の設定
  • Exploreでのフィールド選択・フィルター・ビジュアリゼーション操作

必要なもの

  • Google Cloud プロジェクト（BigQueryが使用可能なもの）
  • Looker インスタンス（Admin権限推奨）
  • 所要時間：約47分

Lookerの3つの構成要素

ビュー（View）

データベースのテーブルと1対1で対応する定義ファイル。テーブルの各列を「ディメンション」または「メジャー」として定義する。

Explore

エンドユーザーに公開するビューの定義。どのビューを・どのJoinで・どのユーザーに公開するかを管理する。

モデル（Model）

接続先データベースと公開するExploreを管理するファイル。プロジェクト全体の「設定書」。

全体フロー

1
BigQuery側の準備データセット作成・サンプルデータ投入・サービスアカウント作成
2
Looker接続設定ConnectionをBigQueryに向けて設定・テスト
3
LookMLプロジェクト作成ウィザードでビュー・モデルファイルを自動生成
4
LookMLの調整メジャー変換・日付型変換・コミット・デプロイ
5
Exploreで可視化フィールド選択・フィルター・ビジュアリゼーション確認

まずBigQuery側でデモ用のデータセットを作成し、サンプルデータを投入します。今回はGA4のオブフスケート済み公開サンプルデータを使用します。

デモ用データセットの作成

  1. BigQueryコンソールを開き、プロジェクト名の横の「＋」をクリックしてデータセットを作成する
  2. データセット名に looker_demo_source と入力する
  3. 「データセットを作成」をクリックする

サンプルデータの投入

BigQueryのクエリエディタで以下のSQLを実行して、ga4_daily_metrics_by_country テーブルを作成します。

WITH UserData AS (
  SELECT
    event_date AS date,
    user_pseudo_id,
    (SELECT value.int_value FROM UNNEST(event_params)
     WHERE key = 'ga_session_id') AS session_id,
    geo.country AS country,
    event_name,
    (SELECT value.string_value FROM UNNEST(event_params)
     WHERE key = 'session_engaged') AS session_engaged_flag,
    (SELECT value.int_value FROM UNNEST(event_params)
     WHERE key = 'engagement_time_msec') AS engagement_time_msec
  FROM `bigquery-public-data.ga4_obfuscated_sample_ecommerce.events_*`
  WHERE PARSE_DATE('%Y%m%d', _TABLE_SUFFIX)
    BETWEEN DATE '2021-01-01' AND DATE '2021-01-31'
)
SELECT
  date,
  COUNT(DISTINCT CASE
    WHEN session_engaged_flag = '1' THEN user_pseudo_id
    WHEN event_name = 'first_visit' THEN user_pseudo_id
    WHEN engagement_time_msec > 0 THEN user_pseudo_id
  END) AS active_users,
  COUNT(DISTINCT CASE
    WHEN event_name = 'first_visit' THEN user_pseudo_id
  END) AS new_users,
  COUNT(DISTINCT CONCAT(user_pseudo_id, CAST(session_id AS STRING))) AS sessions,
  country,
FROM UserData
GROUP BY date, country
ORDER BY date

SQLを実行する際、保存先を looker_demo_source データセット、テーブル名を ga4_daily_metrics_by_country として指定します。

修正1：ディメンションからメジャーへの変換

自動生成では active_users・new_users・sessions がすべて dimension（軸）として定義されています。これらは「合計・集計する数値」なので measure（指標）に変更します。

変更前（自動生成）：

dimension: active_users {
  type: number
  sql: ${TABLE}.active_users ;;
}

変更後（修正）：

measure: active_users {
  type: sum
  sql: ${TABLE}.active_users ;;
}

new_users と sessions も同様に dimension → measure、type: number → type: sum に変更します。

ディメンションとメジャーの違い：
ディメンション（dimension）= 切り口・軸（国、日付、カテゴリなど）
メジャー（measure）= 集計する数値（合計・平均・カウントなど）

修正2：日付データの修正

date 列はBigQueryではINTEGER型（YYYYMMDD形式）で保存されています。Lookerで時系列として扱うために dimension_group に変換します。

変更前（自動生成）：

dimension: date {
  type: number
  sql: ${TABLE}.date ;;
}

変更後（修正）：

dimension_group: date {
  type: time
  timeframes: [date, week, month, year, raw]
  datatype: yyyymmdd
  sql: ${TABLE}.date ;;
}

timeframes を設定することで、Explore上で「日付」「週」「月」「年」単位での集計が選択できるようになります。

変更のコミットと本番反映

1
Save Changes（保存）IDEの「保存」をクリックして編集内容をローカル保存する
2
Validate LookML（バリデーション）「LookMLを検証」をクリックしてエラーがないか確認する
3
Commit Changes & Push（コミット）変更をGitにコミットして記録する
4
Deploy to Production（本番反映）変更を本番環境に適用してエンドユーザーが利用できる状態にする

LookMLのデプロイが完了したら、Explore画面でデータを可視化してみます。

Explore画面の操作

  1. 上部メニューの「探索」→ GA4 Daily Demo（または Ga4 Daily Metrics By Country）をクリック
  2. 左側のフィールド一覧が表示される

フィールドの選択と実行

  1. 左ペインの「Date」グループを展開し、Date Date（日付）をクリック
  2. 「Country」（国）をクリック
  3. 「Sessions」「Active Users」「New Users」をクリック
  4. 右上の「実行」をクリック

行数制限のデフォルトは500行です。データが多い場合は「行数制限」を5,000などに変更して再実行してください。

ビジュアリゼーションの確認

「Country」列が含まれているため、Lookerが自動でGoogleマップビジュアリゼーションを選択します。テーブル形式に変更するには、ビジュアリゼーション選択バーの「テーブル」アイコンをクリックします。

フィルターの使い方

特定の国や期間に絞り込むには、フィールド一覧のフィールドにカーソルを合わせて「フィルタ」アイコンをクリックします。

修正しなかった場合の挙動比較

ディメンションのまま（メジャー修正なし）

active_users 等を dimension のままにしていると、Explore上でグルーピング軸として表示されてしまい、合計値を出せません。

文字列型のまま（日付型修正なし）

date 列を dimension_group に変換せずにいると、YYYYMMDD形式の整数値として表示され、週・月単位での集計ができません。

自動生成のLookMLをそのままデプロイすると、数値列がグルーピング軸として使われてしまい、正しい集計ができません。必ず measure への変換と dimension_group への変換を行ってからデプロイしてください。
