このコースについて

Google Cloud の Cloud Run を使って、コンテナアプリをサーバーレスでデプロイする方法を学ぶハンズオンです。インフラ管理不要で、リクエストが来たときだけ課金される仕組みを体験します。

学習目標

  • Cloud Run の仕組みと用途を理解する
  • Cloud Shell で Python アプリを作成する
  • Docker コンテナをビルドして Artifact Registry に登録する
  • Cloud Run にデプロイしてパブリック URL を取得する
  • トラフィック分割とリビジョン管理を体験する

必要なもの

  • Google Cloud プロジェクト（課金有効化済み）
  • ブラウザ（Cloud Shell を使用するためローカル環境不要）
費用について
このハンズオンは Cloud Shell（無料）と Cloud Run の無料枠（月200万リクエストまで無料）内で完結します。テスト完了後はサービスを削除することを推奨します。

サーバーレスコンテナプラットフォーム

Cloud Run は、コンテナ化されたアプリを インフラ管理なし でデプロイできる Google Cloud のサービスです。

Cloud Run の特徴

特徴内容
サーバーレスサーバーの管理・スケーリングが不要
自動スケールリクエスト数に応じて 0〜N インスタンスに自動調整
コールドスタートリクエストがないときはインスタンス数 0（課金なし）
任意の言語Python・Node.js・Go・Java など Docker で動くものなら何でも
HTTPSデプロイするだけで HTTPS URL が自動付与

Cloud Run が適しているユースケース

  • Web API / REST API のホスティング
  • 定期バッチ処理（Cloud Scheduler との組み合わせ）
  • Webhook 受信（GitHub・Slack などのイベント処理）
  • 社内ツールの軽量 Web アプリ
App Engine との違い
App Engine は言語・フレームワークの制約がありますが、Cloud Run はコンテナが動けば何でも使えます。また Cloud Run の方がスケールダウンの応答が速く、コールドスタートも改善されています。

Cloud Shell を起動する

  1. Google Cloud Console（console.cloud.google.com）を開く
  2. 右上の Cloud Shell アイコン（>_） をクリック
  3. ブラウザ下部にターミナルが起動する

プロジェクトを設定する

gcloud config set project YOUR_PROJECT_ID

必要な API を有効化する

gcloud services enable run.googleapis.com \ artifactregistry.googleapis.com \ cloudbuild.googleapis.com

リージョンを設定する

gcloud config set run/region asia-northeast1
asia-northeast1 は東京リージョンです。レイテンシを最小化するために日本のサービスは東京リージョンを使いましょう。

作業ディレクトリを作成する

mkdir my-cloud-run-app && cd my-cloud-run-app

Python アプリと Dockerfile を作成する

1. Python アプリを作成する

cat > main.py <<'EOF' import os from flask import Flask app = Flask(__name__) @app.route("/") def hello(): name = os.environ.get("NAME", "World") return f"<h1>Hello, {name}!</h1><p>Cloud Run で動いています</p>" if __name__ == "__main__": port = int(os.environ.get("PORT", 8080)) app.run(debug=True, host="0.0.0.0", port=port) EOF

2. requirements.txt を作成する

echo "flask==3.0.3" > requirements.txt

3. Dockerfile を作成する

cat > Dockerfile <<'EOF' FROM python:3.12-slim WORKDIR /app COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . ENV PORT=8080 EXPOSE 8080 CMD ["python", "main.py"] EOF

4. Artifact Registry にリポジトリを作成する

gcloud artifacts repositories create my-repo \ --repository-format=docker \ --location=asia-northeast1

5. コンテナをビルドして push する

PROJECT_ID=$(gcloud config get-value project) docker build -t asia-northeast1-docker.pkg.dev/$PROJECT_ID/my-repo/my-app:v1 . gcloud auth configure-docker asia-northeast1-docker.pkg.dev docker push asia-northeast1-docker.pkg.dev/$PROJECT_ID/my-repo/my-app:v1

デプロイしてサービスを公開する

Cloud Run へデプロイする

PROJECT_ID=$(gcloud config get-value project) gcloud run deploy my-app \ --image asia-northeast1-docker.pkg.dev/$PROJECT_ID/my-repo/my-app:v1 \ --platform managed \ --region asia-northeast1 \ --allow-unauthenticated \ --set-env-vars NAME=AKIBAホールディングス

コマンド実行後、デプロイが完了すると Service URL が表示されます。

Service URL: https://my-app-xxxx-an.a.run.app

ブラウザで確認する

表示された URL をブラウザで開くと、Hello, AKIBAホールディングス! と表示されます。

デプロイオプションの説明

オプション説明
--allow-unauthenticated認証なしでアクセス可（パブリック公開）
--no-allow-unauthenticatedGoogle アカウント認証を要求（社内向け）
--set-env-vars環境変数を設定
--max-instances最大インスタンス数（コスト上限の設定）
--concurrency1インスタンスが同時処理するリクエスト数
社内向けツールは --no-allow-unauthenticated を使い、Google Workspace アカウントを持つ社員だけがアクセスできるよう設定しましょう。

Blue/Green デプロイを体験する

新しいバージョンをデプロイする（トラフィックなし）

PROJECT_ID=$(gcloud config get-value project) gcloud run deploy my-app \ --image asia-northeast1-docker.pkg.dev/$PROJECT_ID/my-repo/my-app:v1 \ --no-traffic \ --tag v2-preview \ --set-env-vars NAME=新バージョン

--no-traffic を指定すると、新リビジョンはデプロイされますがトラフィックは流れません。--tag で専用の URL が発行されるので、テストが可能です。

トラフィックを段階的に切り替える

# v2 に 10% のトラフィックを流す gcloud run services update-traffic my-app \ --to-tags v2-preview=10 # 問題なければ 100% に切り替え gcloud run services update-traffic my-app \ --to-latest

リビジョン一覧を確認する

gcloud run revisions list --service my-app

前のバージョンにロールバックする

gcloud run services update-traffic my-app \ --to-revisions my-app-00001-xxx=100
Blue/Green デプロイのメリット
本番環境への影響を最小化しながら新機能をリリースできます。問題が発生した場合もコマンド1行でロールバックできます。

稼働状況の確認

Cloud Logging でログを確認する

gcloud logging read "resource.type=cloud_run_revision AND resource.labels.service_name=my-app" \ --limit 50 \ --format "table(timestamp,textPayload)"

Cloud Console でメトリクスを確認する

  1. Cloud Console → Cloud Run → my-app
  2. 指標 タブをクリック
  3. 以下のメトリクスを確認できる
メトリクス内容
リクエスト数時間あたりのリクエスト数の推移
レイテンシp50/p95/p99 のレスポンス時間
インスタンス数起動中のコンテナインスタンス数
コンテナ CPUCPU 使用率の推移

クリーンアップ（費用を止める）

# Cloud Run サービスを削除 gcloud run services delete my-app --region asia-northeast1 # Artifact Registry のイメージを削除 gcloud artifacts repositories delete my-repo --location=asia-northeast1
ハンズオン完了後はサービスを削除するか、--max-instances 0 でスケールダウンしておきましょう。
