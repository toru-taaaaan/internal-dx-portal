このコースで学ぶこと

Google Apps ScriptからGemini APIを呼び出し、Gmail・カレンダー・スライドを自動化する方法を学びます。

「AIに日本語で指示するだけで、メール下書き・会議設定・プレゼン作成が自動で完了する」体験ができます。

作るもの

  • 会議設定の自動化：ドキュメントを要約してカレンダーに登録
  • メール下書きの自動生成：スプレッドシートのデータからGmail下書きを作成
  • プレゼン骨組みの自動作成：テーマを渡すとGoogleスライドが完成

前提条件

  • Googleアカウントを持っていること
  • 「GAS 基礎」（CL-002）を完了していること（推奨）
  • 基本的なJavaScriptを読めること

所要時間

約 60 分

Gemini API の利用について
個人のGoogleアカウントで使う場合、無料枠内でもプロンプトデータがGoogleの学習に使われる可能性があります。社内の機密情報は入力しないでください。業務利用にはGoogle Workspace + Vertex AI構成が推奨されます。

APIキーを取得する

Google AI StudioでAPIキーを発行します。

  1. Google AI Studio にアクセス
  2. 「APIキーを作成」をクリック
  3. 「新しいプロジェクトでAPIキーを作成」を選択
  4. 表示されたAPIキーをコピーして保存しておく

APIキーは秘密情報です
コードに直接書き込まず、Apps Scriptのプロパティに保存します（次のステップで設定します）。

Apps Script プロジェクトを作成する

  1. script.google.com にアクセス
  2. 「新しいプロジェクト」をクリック
  3. プロジェクト名を「Gemini Workspace 自動化」に変更
  4. 左側ファイル一覧で「コード.gs」を右クリック →「名前を変更」→ utils に変更

APIキーをスクリプトプロパティに保存する

  1. 左側の「プロジェクトの設定」（歯車アイコン）をクリック
  2. 「スクリプトのプロパティ」セクションで「プロパティを追加」をクリック
  3. プロパティ名: GOOGLE_API_KEY、値: 先ほどコピーしたAPIキーを入力
  4. 「保存」をクリック

なぜプロパティに保存するか
スクリプトプロパティはコードと分離して管理されます。GitHubに公開してもAPIキーが漏れません。また、チームで共有するときに各自が自分のキーを設定できます。

基本的な呼び出し関数を作る

utils.gs に以下のコードを入力してください：

// Gemini API の基本呼び出し関数
function callGemini(prompt, temperature = 0) {
  var apiKey = PropertiesService.getScriptProperties().getProperty("GOOGLE_API_KEY");
  var url = "https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=" + apiKey;

  var payload = {
    "contents": [
      {
        "parts": [
          { "text": prompt }
        ]
      }
    ],
    "generationConfig": {
      "temperature": temperature
    }
  };

  var options = {
    "method": "POST",
    "contentType": "application/json",
    "payload": JSON.stringify(payload)
  };

  var response = UrlFetchApp.fetch(url, options);
  var data = JSON.parse(response.getContentText());
  return data.candidates[0].content.parts[0].text;
}

動作確認

テスト用の関数を utils.gs の末尾に追加します：

// テスト用
function testGemini() {
  var result = callGemini("Google Apps Script について3行で説明してください。");
  Logger.log(result);
}
  1. 保存（Ctrl+S）
  2. testGemini を選択して実行（▶）
  3. 「表示」→「ログ」でGeminiの回答を確認

temperature とは
0に近いほど決定論的（毎回同じ回答）、1に近いほど創造的（回答にバラつきが出る）。 事実の要約や構造化には0、アイデア出しや文章生成には0.5〜1を使います。

JSON形式で回答させる

プログラムで結果を扱うには、JSON形式で返させると便利です：

function testGeminiJson() {
  var prompt = `以下の情報から会議の件名と要約を作成してください。
必ず次のJSON形式で返してください:
{"subject": "件名", "summary": "要約（100文字以内）"}

情報: 来週の月曜日、営業部と開発部が集まって新製品のロードマップを議論する。
参加者は10名程度。オンライン開催。`;

  var result = callGemini(prompt, 0);

  // JSON部分を抽出してパース
  var jsonMatch = result.match(/\{[\s\S]*\}/);
  if (jsonMatch) {
    var parsed = JSON.parse(jsonMatch[0]);
    Logger.log("件名: " + parsed.subject);
    Logger.log("要約: " + parsed.summary);
  }
}

ドキュメントを要約してカレンダーに登録する

Googleドライブにあるファイルの内容を読み取り、Geminiで要約し、カレンダーに会議を登録します。

ステップ 1：テスト用テキストを用意する

  1. Googleドライブで新しいドキュメントを作成
  2. 「来週の新製品企画会議について」という内容を数段落書いておく（日時・議題・参加者など）
  3. ファイルのURLからIDをコピーしておく（https://docs.google.com/document/d/【このID部分】/edit）

ステップ 2：会議設定関数を書く

新しいファイル meeting.gs を作成し、以下を入力：

function setupMeeting(fileId) {
  // ドキュメントの内容を取得
  var doc = DocumentApp.openById(fileId);
  var text = doc.getBody().getText();

  // Geminiで要約と日程を抽出
  var prompt = `以下のドキュメントから会議情報を抽出してください。
必ず次のJSON形式で返してください:
{
  "title": "会議のタイトル",
  "summary": "会議の目的・内容（200文字以内）",
  "date": "YYYY-MM-DD形式の日付（不明な場合は来週月曜日の日付）",
  "duration": 60
}

ドキュメント:
${text}`;

  var result = callGemini(prompt, 0);
  var jsonMatch = result.match(/\{[\s\S]*\}/);
  if (!jsonMatch) {
    Logger.log("JSONの抽出に失敗しました");
    return;
  }

  var meeting = JSON.parse(jsonMatch[0]);
  Logger.log("会議情報: " + JSON.stringify(meeting));

  // カレンダーに登録
  var calendar = CalendarApp.getDefaultCalendar();
  var startDate = new Date(meeting.date + "T10:00:00");
  var endDate = new Date(startDate.getTime() + meeting.duration * 60000);

  var event = calendar.createEvent(
    meeting.title,
    startDate,
    endDate,
    { description: meeting.summary }
  );

  Logger.log("カレンダーに登録しました: " + event.getTitle());
  return event;
}

// テスト実行用（fileIdは実際のIDに変更してください）
function testSetupMeeting() {
  var fileId = "YOUR_DOCUMENT_ID_HERE"; // ← 変更してください
  setupMeeting(fileId);
}

ステップ 3：実行と確認

  1. fileId を実際のドキュメントIDに変更
  2. testSetupMeeting を実行
  3. Googleカレンダーに会議が登録されているか確認

スプレッドシートのデータからメール下書きを作る

顧客リストや報告データが入ったスプレッドシートを元に、Geminiが文面を生成してGmailの下書きを作成します。

ステップ 1：サンプルスプレッドシートを用意する

新しいスプレッドシートを作成し、以下のデータを入力してください：

A列: 宛先氏名B列: 会社名C列: 件名D列: 要点
山田 太郎株式会社サンプル先日のご提案について見積もりを送付済み。来週中に返事をもらいたい。

ステップ 2：メール生成関数を書く

新しいファイル email.gs を作成し、以下を入力：

function draftEmail() {
  var sheet = SpreadsheetApp.getActiveSheet();
  var lastRow = sheet.getLastRow();

  // ヘッダー行を除いてデータを読む
  for (var i = 2; i <= lastRow; i++) {
    var name    = sheet.getRange(i, 1).getValue();
    var company = sheet.getRange(i, 2).getValue();
    var subject = sheet.getRange(i, 3).getValue();
    var points  = sheet.getRange(i, 4).getValue();

    if (!name) continue;

    // Geminiでメール本文を生成
    var prompt = `以下の情報をもとに、ビジネスメールの本文を作成してください。
件名: ${subject}
宛先: ${company} ${name} 様
要点: ${points}

条件:
- 丁寧なビジネス敬語を使う
- 本文のみを出力する（件名・宛先は不要）
- 200〜300文字程度`;

    var body = callGemini(prompt, 0.3);

    // Gmail の下書きを作成
    GmailApp.createDraft(
      "",  // 宛先メールアドレス（実際は入力してください）
      subject,
      body
    );

    Logger.log(name + " さんへの下書きを作成しました");
  }

  SpreadsheetApp.getUi().alert("下書きの作成が完了しました。Gmailで確認してください。");
}

ステップ 3：カスタムメニューから実行できるようにする

utils.gs の先頭に onOpen() 関数を追加します：

function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu("Gemini 自動化")
    .addItem("メール下書きを作成", "draftEmail")
    .addToUi();
}

  2. メニューに「Gemini 自動化」が表示されることを確認
  3. 「メール下書きを作成」をクリック
  4. Gmailの「下書き」フォルダに生成されたメールを確認

テーマを渡すとスライドが完成する

Geminiにテーマを渡すとスライドの構成を考えてくれます。その結果をGoogleスライドに自動で流し込みます。

スライド生成関数を書く

新しいファイル slides.gs を作成し、以下を入力：

var NUM_SLIDES = 5; // 生成するスライド数

function createDeck(topic) {
  // Geminiでスライド構成を生成
  var prompt = `「${topic}」について、${NUM_SLIDES}枚のプレゼンテーションスライドの構成を作成してください。
必ず次のJSON形式で返してください:
{
  "title": "プレゼンのタイトル",
  "slides": [
    {"heading": "スライドタイトル", "bullets": ["箇条書き1", "箇条書き2", "箇条書き3"]},
    ...
  ]
}`;

  var result = callGemini(prompt, 0.5);
  var jsonMatch = result.match(/\{[\s\S]*\}/);
  if (!jsonMatch) {
    Logger.log("JSONの抽出に失敗しました");
    return;
  }

  var deckData = JSON.parse(jsonMatch[0]);
  Logger.log("スライド構成: " + JSON.stringify(deckData));

  // Google スライドを作成
  var presentation = SlidesApp.create(deckData.title);

  deckData.slides.forEach(function(slideData, index) {
    var slide;
    if (index === 0) {
      // 最初のスライドは既存のものを使う
      slide = presentation.getSlides()[0];
    } else {
      slide = presentation.appendSlide(SlidesApp.PredefinedLayout.TITLE_AND_BODY);
    }

    // タイトルと本文を設定
    var shapes = slide.getShapes();
    shapes[0].getText().setText(slideData.heading);
    if (shapes[1]) {
      shapes[1].getText().setText(slideData.bullets.join("\n"));
    }
  });

  Logger.log("スライドを作成しました: " + presentation.getUrl());
  return presentation.getUrl();
}

// テスト実行用
function testCreateDeck() {
  var url = createDeck("社内DX推進の意義と具体的な取り組み");
  Logger.log("完成URL: " + url);
}

実行してみる

  1. 保存後、testCreateDeck を実行
  2. ログに表示されたURLをクリック
  3. Googleスライドが自動作成されていることを確認
  4. スライドの内容を見て、手動で仕上げる
