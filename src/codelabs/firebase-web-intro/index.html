Firebase とは

Firebase は Google が提供するモバイル・Web アプリ開発プラットフォームです。データベース・認証・ホスティングなどの機能がセットになっており、バックエンドのサーバーを自分で用意しなくてもリアルタイムに動く Web アプリを作れます。

「サーバーなしでリアルタイムデータベースが使えるサービス」とイメージしてください。複数ユーザーが同時に書き込んでも、全員の画面に即座に反映されます。

このコースで作るもの

イベント参加者向けの ゲストブック（リアルタイムチャット） を作ります。

  • Google アカウントまたはメールで参加登録（Firebase Authentication）
  • メッセージをリアルタイムで送受信（Cloud Firestore）
  • セキュリティルールで不正アクセスを防止

このコースで学ぶこと

  • Firebase プロジェクトの作成と設定
  • Firebase Authentication でユーザー認証を実装
  • Cloud Firestore へのデータ書き込み・読み込み
  • リアルタイムリスナーでの即時反映
  • Firebase Security Rules でのアクセス制御

前提条件

  • Google アカウントを持っていること
  • HTML と JavaScript の基本を知っていること（必須ではないが推奨）
  • ブラウザ（Chrome 推奨）があること

チェックポイントを使う

Firebase コンソールでプロジェクトを作成

  1. console.firebase.google.com にアクセス
  2. 「プロジェクトを追加」 をクリック
  3. プロジェクト名を入力（例：firebase-web-codelab）
  4. 「続行」をクリック
  5. Google Analytics は今回は無効にして「プロジェクトを作成」
  6. プロジェクトの作成が完了したら「続行」

Authentication を有効化する

  1. 左メニューの「構築」→「Authentication」をクリック
  2. 「始める」をクリック
  3. 「ログイン方法」タブを選択
  4. 「メール/パスワード」 をクリック
  5. 「有効にする」トグルをオンにして「保存」

Cloud Firestore を作成する

  1. 左メニューの「構築」→「Firestore Database」をクリック
  2. 「データベースを作成」をクリック
  3. データベース ID はデフォルト（(default)）のままで「次へ」
  4. ロケーションを選択（asia-northeast1 = 東京 推奨）
  5. 「次へ」をクリック
  6. 「テストモードで開始する」 を選択して「作成」

テストモードについて
テストモードは 30 日間のみ有効で、誰でも読み書きができる状態です。後のステップでセキュリティルールを設定します。

Web アプリを Firebase に登録する

  1. Firebase コンソールの「プロジェクトの概要」を開く
  2. 画面中央の </>（Web）アイコンをクリック
  3. アプリのニックネームを入力（例：Web App）
  4. 「Firebase Hosting の設定」は今回チェックしない
  5. 「アプリを登録」をクリック
  6. 表示される設定オブジェクトをコピーしておく

設定コードを index.js に追加する

StackBlitz の index.js を開き、コメント // Add Firebase project configuration object here の下に、コピーした設定を貼り付けます。

import { initializeApp } from 'firebase/app';
import {} from 'firebase/auth';
import {} from 'firebase/firestore';
import * as firebaseui from 'firebaseui';

// ここにコピーした設定を貼り付ける
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.firebasestorage.app",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

// Firebase を初期化
initializeApp(firebaseConfig);

認証の仕組みを追加する

FirebaseUI という UI ライブラリを使って、サインインフォームを自動生成します。

ステップ 1：auth のインポートを更新する

import {
  getAuth,
  EmailAuthProvider,
  signOut,
  onAuthStateChanged
} from 'firebase/auth';

ステップ 2：auth の参照を保存する

initializeApp(firebaseConfig); の後に追加します。

const auth = getAuth();

ステップ 3：FirebaseUI を初期化する

main() 関数の中に追加します。

const ui = new firebaseui.auth.AuthUI(auth);

const uiConfig = {
  credentialHelper: firebaseui.auth.CredentialHelper.NONE,
  signInOptions: [
    EmailAuthProvider.PROVIDER_ID,
  ],
  callbacks: {
    signInSuccessWithAuthResult: function() {
      return false;
    },
  },
};

ステップ 4：RSVP ボタンを HTML に追加する

index.html の event-details-container 内に追加します。

<button id="startRsvp">RSVP</button>

ステップ 5：ボタンのリスナーを設定する

// RSVP ボタンのクリックイベント
startRsvpButton.addEventListener('click', () => {
  if (auth.currentUser) {
    signOut(auth);
  } else {
    ui.start('#firebaseui-auth-container', uiConfig);
  }
});

// サインイン状態の変化を監視
onAuthStateChanged(auth, user => {
  if (user) {
    startRsvpButton.textContent = 'LOGOUT';
  } else {
    startRsvpButton.textContent = 'RSVP';
  }
});

  1. 「RSVP」ボタンをクリックするとサインインフォームが表示される
  2. 任意のメールアドレスでサインイン
  3. ボタンが「LOGOUT」に変わることを確認
  4. Firebase コンソールの「Authentication」→「ユーザー」タブにユーザーが追加される

ステップ 4：フォーム送信のリスナーを追加する

form.addEventListener('submit', async e => {
  e.preventDefault();
  // Firestore の guestbook コレクションにドキュメントを追加
  await addDoc(collection(db, 'guestbook'), {

リアルタイムリスナーとは

Firestore の onSnapshot を使うと、データベースの変更が即座にアプリに反映されます。ページを再読み込みしなくても、他のユーザーが書き込んだメッセージがリアルタイムで表示されます。

メッセージ一覧を表示する

ステップ 1：HTML に表示エリアを追加する

<section id="guestbook-container">
  <h2>メッセージを残す</h2>
  <form id="leave-message">...</form>
  <!-- メッセージ一覧を表示するエリア -->
  <section id="guestbook"></section>
</section>

ステップ 2：インポートを更新する

import {
  getFirestore,
  addDoc,
  collection,
  query,
  orderBy,
  onSnapshot
} from 'firebase/firestore';

ステップ 3：リアルタイムリスナーを追加する

// タイムスタンプの降順でクエリを作成
const q = query(
  collection(db, 'guestbook'),
  orderBy('timestamp', 'desc')
);

// データが変更されるたびに自動的に呼ばれる
onSnapshot(q, snaps => {
  // 表示エリアをクリア
  guestbook.innerHTML = '';
  // 各ドキュメントを表示
  snaps.forEach(doc => {
    const entry = document.createElement('p');
    entry.textContent = doc.data().name + ': ' + doc.data().text;
    guestbook.appendChild(entry);
  });
});

動作確認

  1. アプリをリロードすると、過去のメッセージが表示される
  2. 新しいメッセージを送信すると即座に一覧に追加される
  3. ブラウザの別タブで同じ URL を開いて、両方に同時に反映されることを確認

なぜセキュリティルールが必要か

テストモードで作成した Firestore は、誰でも読み書きができます。本番環境では、ログインしているユーザーだけが書き込めるよう制限する必要があります。

ルールを設定する

  1. Firebase コンソール →「Firestore Database」→「ルール」タブを開く
  2. 以下のルールに書き換えて「公開」をクリック
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /guestbook/{entry} {
      // 認証済みユーザーのみ読める
      allow read: if request.auth.uid != null;

      // 自分の userId でのみ書き込める（なりすまし防止）
      // かつ必須フィールドがすべて揃っていること
      allow create:
        if request.auth.uid == request.resource.data.userId
          && "name" in request.resource.data
          && "text" in request.resource.data
          && "timestamp" in request.resource.data;
    }
  }
}

ルールの読み方

ルール 意味
allow read: if request.auth.uid != null ログインしているユーザーのみ読める
request.auth.uid == request.resource.data.userId 自分の UID でしか書き込めない
"name" in request.resource.data name フィールドが必ず含まれること

リスナーのサブスクリプション管理

サインアウトしたらリスナーを停止し、サインインしたら再起動するよう onAuthStateChanged を更新します。

let guestbookListener = null;

function subscribeGuestbook() {
  const q = query(collection(db, 'guestbook'), orderBy('timestamp', 'desc'));
  guestbookListener = onSnapshot(q, snaps => {
    guestbook.innerHTML = '';
    snaps.forEach(doc => {
      const entry = document.createElement('p');
      entry.textContent = doc.data().name + ': ' + doc.data().text;
      guestbook.appendChild(entry);
    });
  });
}

function unsubscribeGuestbook() {
  if (guestbookListener != null) {
    guestbookListener();
    guestbookListener = null;
  }
}

onAuthStateChanged(auth, user => {
  if (user) {
    startRsvpButton.textContent = 'LOGOUT';
    guestbookContainer.style.display = 'block';
    subscribeGuestbook(); // リスナー開始
  } else {
    startRsvpButton.textContent = 'RSVP';
    guestbookContainer.style.display = 'none';
    unsubscribeGuestbook(); // リスナー停止
  }
});
