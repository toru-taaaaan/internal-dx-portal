VPN構成の概要

  • VarioSecure社 VSR601J/VSR402J によるハブ&スポーク構成
  • 本社 VSR601J がハブ、他7拠点がスポーク
  • 全拠点の Internet 通信が本社を経由（ボトルネック）
  • ISP: USEN GATE 02（全8回線）/ NTT フレッツ光

大阪の部分的LBO

  • ルーターを VSR601J に変更し、GWS 向けの LBO を実現
  • Google Workspace のトラフィックのみ大阪から直接 Internet へ
  • 業務通信のメインは依然として本社 VPN 経由
  • 他拠点への LBO 展開は「1台ずつ個別設定」が必要（USEN回答）

VPN依存システム一覧（セコム以外も多数）

  • SECOM入退室管理: TRIISRV（本社）↔ 全9拠点TRIIインターフェース
  • Active Directory: PDCSVR/BDCSVR（本社）↔ 全拠点PC認証・GPO
  • ファイルサーバー: ADSHARESVR（本社 .1.15）↔ 各Type A拠点からアクセス
  • IoTファイルサーバー: IOTSVR（橋本 .30.15/.16）＋ QNAP (.30.10)
  • TENSUITE: 勤怠管理（本社 .1.12/.14）↔ 全拠点から参照
  • 複合機: RICOH IM C4500等 → AD参照（SMTP/LDAP/SMB）

Cato Socket/Client 判断のポイント

  • Socket が必要な条件: Client をインストールできない機器が Cato NW経由の通信を必要とする場合
  • Secom入退室機器: Client不可 → VPN or Socket 必須
  • ファイルサーバー/TENSUITE: サーバーOS → Socket または クラウド移行が必要
  • 業務PC: Client インストール可 → Socket 不要
  • 複合機: ローカルLAN完結なら Socket 不要（ただしAD参照がVPN経由の場合は要検討）

VPN全廃に向けたロードマップ

  • Secom入退室: フルクラウド型に移行すればVPN不要（別途検討中）
  • AD: Entra ID（旧Azure AD）移行で拠点間VPN不要に
  • ファイルサーバー: SharePoint / Azure Files 等への集約でVPN不要に
  • TENSUITE: クラウド版への移行 or VPN経由の維持を判断
  • 全て移行完了まではVPN残存 → Cato導入時にSocket台数に影響
