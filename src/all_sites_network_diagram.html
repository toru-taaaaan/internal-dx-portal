拠点分類の定義と前提

1. カテゴリ1: 本社（築地拠点）

  • 全社VarioSecure VPN網の終端（HUB拠点）でありつつ、業務通信を集約するCato Socketも併設される中心拠点です。
  • 「すべての拠点にVarioSecureを残すが、SECOMの入退室管理機器以外は一切つながない」という強力な原則を維持し、社内ネットワーク（Cato）とは完全に分離されます。

2. カテゴリ2: 主要業務拠点（橋本 / 多摩 / 大阪）

  • PCなどの業務通信（Cato Socket）と、SECOM通信（VarioSecure）をハブやスイッチで混在させず完全に分離している拠点です。
  • 本社にもCatoが導入されているため、この構成で社内・社外のすべての通信がシンプルに成立します。

3. カテゴリ3: その他業務拠点（京橋 / 鬼怒川）

  • カテゴリ2と同様に、業務PCのネットワークと、SECOMしか繋がっていないVarioSecureネットワークを分離しています。

4. カテゴリ4: 勤怠限定拠点（名古屋 / 和歌山 等）

  • 勤怠打刻端末等の通信と、SECOM通信（VarioSecure）を完全に分離しています。
  • VarioSecureには他拠点と同様にSECOM機器のみが接続されます。打刻端末は通常のインターネットルーター等を使用します。

5. カテゴリ5: バディネット拠点（福岡 等）

  • インターネット回線からまずルーターやハブを経由し、そこから「バディネット独自の社内ネットワーク」と「VarioSecure」の2系統に分岐する特殊構成です。
  • VarioSecureの配下にはSECOM機器のみが存在しており、SECOM通信専用回線としてVarioSecureが機能しています。

各拠点 詳細ネットワーク構成図（全10拠点）

各拠点に設置されている全機器（ファイルサーバー、複合機、バディネット用ルーター等）を詳細化し、全10拠点を1つずつ網羅しています。Cato導入時の Socket/Client判定基準も含みます。
