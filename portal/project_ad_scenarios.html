【基礎知識】Active Directory と Entra ID の役割の違い

A Active Directory (AD) の役割

「社内ネットワークという境界（箱）」を守る

  • ● PCのログイン管理（ドメイン参加）
  • ● ファイルサーバー等の社内リソース利用許可
  • ● PC設定の強制（グループポリシー）
  • ※USB使用禁止、パスワードの桁数強制、デスクトップ壁紙の固定など

E Microsoft Entra ID の役割

「インターネット上のユーザー（ID）」を守る

  • ● クラウドサービス（SaaS）へのログイン管理
  • ● 多要素認証（MFA）によるなりすまし防止
  • ● 条件付きアクセス（特定の場所・端末からのみ許可）
  • ※スマホや私物PCからのアクセス制限、海外からのログイン遮断など

決定的な違い：クラウド化すれば同じ管理ができるのか？

結論から言うと、「Entra IDだけ」では、従来のADのようなきめ細やかなPC管理（グループポリシー等）はできません。

● Entra ID単体で「できなくなること」

  • ・ファイルサーバーへの自動ログイン（Kerberos認証）
  • ・PC内部設定の詳細な一括制御（GPO）
  • ・インターネットがない環境でのPCログイン

これらをクラウド上で補うには、Entra IDに加えて 「Microsoft Intune」 という別のサービスが必要になります。 今回のプロジェクトで「ADをクラウドにリフトする」のは、現在のきめ細やかな管理機能をそのまま使い続けるための最も確実な手段です。
